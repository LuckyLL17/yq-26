## 2. 卡牌完整生命周期

卡牌在游戏中经历五个阶段：**牌库 → 手牌 → 使用 → 弃牌堆 → 洗回牌库**

```
┌─────────────┐
│    牌库     │ 初始 20 张，洗牌后随机
└─────┬───────┘
      │ 抽牌（从牌库顶 pop）
      ↓
┌─────────────┐
│    手牌     │ 最多 5 张
└─────┬───────┘
      │ 使用卡牌
      ↓
┌─────────────┐
│   弃牌堆    │ 使用后进入
└─────┬───────┘
      │ 牌库空时自动洗入
      ↓
┌─────────────┐
│    牌库     │ 重新洗牌后回到牌库
└─────────────┘
```

### 2.1 阶段一：牌库 (Deck)

#### 初始化

游戏开始时生成初始牌库，流程如下：

1. 调用 `generateRandomDeck()` 生成 20 张卡牌类型列表 [config.ts](file:///Volumes/ExMac/traeProject/全站1/yq-26/src/game/config.ts#L361-L394)
2. 按稀有度权重随机抽取卡牌类型
3. 通过 `shuffleDeck()` 使用 Fisher-Yates 算法洗牌 [config.ts](file:///Volumes/ExMac/traeProject/全站1/yq-26/src/game/config.ts#L396-L403)

```typescript

  // 按稀有度权重随机抽取...

```

初始状态创建在 [store.ts](file:///Volumes/ExMac/traeProject/全站1/yq-26/src/game/store.ts#L48-L67) 的 `createInitialState()` 中：

```typescript

    hand.push(deck.pop()!);
  }
  // ...

```

**牌库特点**：
- 初始 20 张
- 按稀有度权重随机生成
- 使用 Fisher-Yates 洗牌算法打乱顺序

### 2.2 阶段二：手牌 (Hand)

#### 进入手牌的方式

**1. 游戏开始抽牌**

- 初始抽取 5 张手牌（`MAX_HAND_SIZE = 5`）

**2. 波次开始抽牌**

每波开始时调用 `drawCards(2)` 抽取 2 张 [store.ts](file:///Volumes/ExMac/traeProject/全站1/yq-26/src/game/store.ts#L783-L784)

```typescript
startWave: () => {
  // ...
  get().drawCards(2);
  // ...
}
```

#### 抽牌逻辑

抽牌逻辑在 `drawCards()` 函数 [store.ts](file:///Volumes/ExMac/traeProject/全站1/yq-26/src/game/store.ts#L664-L683)：

```typescript
drawCards: (count) => {
  const state = get();
  let { deck, hand, discardPile } = state;

  for (let i = 0; i < count; i++) {
    // 手牌已满则停止
    if (hand.length >= MAX_HAND_SIZE) break;

    // 牌库空了就洗入弃牌堆
    if (deck.length === 0) {
      if (discardPile.length === 0) break;
      deck = shuffleDeck([...discardPile]);
      discardPile = [];
    }

    // 从牌库顶抽一张
    const card = deck[deck.length - 1];
    deck = deck.slice(0, -1);
    hand = [...hand, card];
  }

  set({ deck, hand, discardPile });
}
```

**手牌特点**：
- 最大手牌数：5 张
- 从牌库顶部抽牌
- 牌库空时自动将弃牌堆洗回牌库

### 2.4 阶段四：弃牌堆 (Discard Pile)

#### 进入弃牌堆的情况

1. **使用卡牌后**：卡牌从手牌移至弃牌堆
2. **波次奖励未选择**：未选中的奖励卡全部进入弃牌堆

#### 波次奖励机制

每波敌人清完后，玩家可从 3 张奖励卡中选择 1 张加入牌库 [store.ts](file:///Volumes/ExMac/traeProject/全站1/yq-26/src/game/store.ts#L793-L855)

```typescript
// 选择奖励卡：选一张加入牌库，其余进入弃牌堆
collectWaveReward: (cardId) => {
  const card = state.waveRewardCards.find(c => c.id === cardId);
  let newDeck = [...state.deck, card];
  newDeck = shuffleDeck(newDeck);
  const remainingCards = state.waveRewardCards.filter(c => c.id !== cardId);
  const newDiscardPile = [...state.discardPile, ...remainingCards];
}

// 跳过奖励：全部进入弃牌堆
skipWaveReward: () => {
  const newDiscardPile = [...state.discardPile, ...state.waveRewardCards];
}
```

波次奖励卡牌生成在 `generateWaveRewardCards()` [store.ts](file:///Volumes/ExMac/traeProject/全站1/yq-26/src/game/store.ts#L685-L740)，随着波数增加，稀有卡概率会提升。

#### 洗牌算法

Fisher-Yates 洗牌算法 [config.ts](file:///Volumes/ExMac/traeProject/全站1/yq-26/src/game/config.ts#L396-L403)

```typescript
export function shuffleDeck<T>(deck: T[]): T[] {
  const shuffled = [...deck];
  for (let i = shuffled.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [shuffled[i], shuffled[j]] = [shuffled[j], shuffled[i]];
  }
  return shuffled;
}
```

## 3. 卡牌交互链路

从选择卡牌到释放效果的完整交互流程。
